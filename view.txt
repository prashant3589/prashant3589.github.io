Story Title

Resolve QA Automation Pipeline Failures Caused by Agent Disk Space Exhaustion

Description

The QA automation pipeline is intermittently failing during execution because the build agent runs out of available disk space. The issue occurs due to repeated source code checkouts and accumulation of workspace artifacts on the agent machine. When disk usage reaches its limit, pipeline tasks fail, impacting test execution, release timelines, and overall pipeline reliability.

This story aims to identify, mitigate, and permanently resolve disk space exhaustion on QA automation agents by implementing proper workspace cleanup, optimizing checkout behavior, and/or improving agent configuration to ensure consistent and stable pipeline runs.

Solution works consistently across all QA automation pipelines using the same agent pool.

No manual intervention is required to free disk space on the agent.
