Title

Implement Flyway-based Database Migration Approach using Visual Studio Integration

📄 Description

As a developer,
I want to generate Flyway migration scripts directly from Visual Studio using external tools,
So that I can capture database changes from DEV environment and maintain version-controlled, incremental migrations without changing existing EF/DACPAC workflow.


🛠️ Scope
Configure Flyway CLI on developer machines / build agents
Integrate Flyway commands into Visual Studio (External Tools)
Enable developers to:
Run flyway diff
Run flyway generate
Run flyway validate
Standardize migration folder structure
Ensure compatibility with existing EF/DACPAC workflow
🧩 Implementation Details
🔹 Visual Studio Integration
Configure External Tools:
Flyway Info
Flyway Diff
Flyway Generate
Flyway Validate
🔹 Flyway Configuration
Use flyway.toml for:
Database connection
Migration folder path
Environment configuration
🔹 Migration Strategy
Use incremental versioned scripts

Follow naming convention:

VYYYYMMDD_HHMM__description.sql
🔹 Developer Workflow
Pull latest develop
Apply DB changes in DEV
Run Flyway diff (DEV → migrations)
Generate new migration script
Commit only new migration
Raise PR
✅ Acceptance Criteria
✔ Functional
 Flyway CLI is installed and accessible via environment variables
 Flyway commands can be executed from Visual Studio External Tools
 Developer can generate migration using:
flyway diff
flyway generate
 Migration script is created in configured migrations folder
 Migration naming follows timestamp-based convention
 flyway validate runs successfully
